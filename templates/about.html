MCA Final Project

Explainable Multilingual SMS Spam Detection

Using Machine Learning with Confidence Scoring and Translation-Based Support — a practical, interpretable, and multilingual spam detection system that goes far beyond existing English-only binary classifiers.

5 Research Novelties
1

Explainable AI (XAI) — Not a Black Box

Most classifiers give only SPAM/HAM. SpamShield uses Logistic Regression coefficients to pinpoint exactly which words and patterns triggered the classification. This improves user trust, transparency, and is rare in student-level projects.

2

Multilingual + Mixed-Language Support

Supports English, Tamil, Hindi, Telugu, Tanglish, Hinglish and 7+ languages via automatic language detection (langdetect) and translation (googletrans). Makes the system practically useful for Indian users — a major gap in existing work.

3

Calibrated Confidence Scoring with Risk Levels

Instead of a binary output, users receive a precise probability score (e.g. 94.3%) and a risk tier (Critical / High / Medium / Safe). SVM is wrapped in CalibratedClassifierCV to produce true probability estimates.

4

Dual TF-IDF + Statistical Meta-Features

Combines character n-grams (captures Tanglish/Hinglish patterns), word n-grams (captures phrase-level spam), and 10 statistical meta-features (message length, caps ratio, URL presence, currency symbols, etc.) — a richer representation than any single vectoriser.

5

Complete Web System with Prediction History

A production-grade Flask application with 5 pages, real-time prediction, language detection, session-based history tracking, and visual performance dashboards — not just a Jupyter notebook.
